🗺️ ภาพรวมระบบ

ระบบนี้ใช้ Supabase เป็นฐานข้อมูลบนคลาวด์ฟรี เก็บข้อมูลใบเสร็จค่าเช่าห้องและข้อมูลผู้เช่า รองรับการ Login ด้วย Email/Password และมีระบบป้องกันข้อมูลด้วย Row Level Security (RLS)

💡
Supabase มีแพลนฟรี (Free Tier) รองรับฐานข้อมูล 500MB เพียงพอสำหรับการจัดการห้องเช่าขนาดเล็ก-กลาง
สิ่งที่ต้องการรายละเอียดค่าใช้จ่าย
บัญชี Supabaseสมัครด้วย Email หรือ GitHubฟรี
Supabase Project1 Project ต่อหอพักฟรี
ตาราง room_recordsเก็บประวัติบิลทั้งหมดฟรี
ตาราง rooms_dataเก็บข้อมูลห้องและผู้เช่าฟรี
Supabase Auth Userบัญชีสำหรับ Loginฟรี

1️⃣ สมัครบัญชี Supabase

  1. เปิดเว็บไซต์ Supabase

    ไปที่ supabase.com แล้วคลิกปุ่ม "Start your project" หรือ "Sign Up"

  2. เลือกวิธีสมัคร

    แนะนำสมัครด้วย GitHub (ง่ายที่สุด) หรือใช้ Email ธรรมดาก็ได้

  3. ยืนยันอีเมล

    ถ้าสมัครด้วย Email จะมีลิงก์ยืนยันส่งมาให้ กดยืนยันก่อนเข้าใช้งาน

2️⃣ สร้าง Project ใหม่

  1. คลิก "New Project"

    หลัง Login แล้วจะเห็น Dashboard คลิกปุ่ม "New Project"

  2. กรอกชื่อ Project

    ตั้งชื่อตามชื่อหอพักของคุณ เช่น my-apartment

  3. ตั้งรหัสผ่านฐานข้อมูล

    กรอก Database Password (เก็บไว้ด้วย แต่ระบบนี้ไม่ได้ใช้โดยตรง)

  4. เลือก Region

    เลือก Southeast Asia (Singapore) เพื่อให้โหลดเร็วในไทย

  5. คลิก "Create new project"

    รอประมาณ 1-2 นาที ระบบจะสร้าง Project ให้อัตโนมัติ

⚠️
Free Tier ของ Supabase สร้างได้ 2 Projects ต่อ 1 บัญชี — และ "Database Password" ที่ตั้งในขั้นตอนนี้ ไม่ใช่ รหัสผ่านที่ใช้ Login เข้าเว็บออกใบเสร็จ

3️⃣ สร้างตารางเก็บข้อมูล

ไปที่เมนู SQL Editor ทางซ้ายมือ แล้วรัน SQL ด้านล่างนี้ ต้องรัน 2 ส่วน แยกกัน

ส่วนที่ 1 — ตาราง room_records (เก็บประวัติบิล)

-- ตารางเก็บประวัติใบเสร็จทั้งหมด
CREATE TABLE room_records (
    id            bigint        GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    timestamp     timestamptz   DEFAULT now(),
    month         text,
    room          text,
    rent          float4        DEFAULT 0,
    water_before  float4        DEFAULT 0,
    water_after   float4        DEFAULT 0,
    water_unit    float4        DEFAULT 0,
    water_total   float4        DEFAULT 0,
    elec_before   float4        DEFAULT 0,
    elec_after    float4        DEFAULT 0,
    elec_unit     float4        DEFAULT 0,
    elec_total    float4        DEFAULT 0,
    internet      float4        DEFAULT 0,
    discount      float4        DEFAULT 0,
    grand_total   float4        DEFAULT 0
);

ส่วนที่ 2 — ตาราง rooms_data (เก็บข้อมูลห้องและผู้เช่า)

-- ตารางเก็บข้อมูลห้องและผู้เช่าปัจจุบัน
CREATE TABLE rooms_data (
    id            bigint        GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    room_number   text          NOT NULL,
    tenant_name   text,
    phone         text,
    deposit       float4        DEFAULT 0,
    base_rent     float4        DEFAULT 0,
    fine          float4        DEFAULT 0,
    paid_status   boolean       DEFAULT false,
    last_total    float4,
    last_month    text,
    last_water    float4        DEFAULT 0,
    last_elec     float4        DEFAULT 0
);
✅
ถ้ารันสำเร็จจะเห็นข้อความ "Success. No rows returned" ที่ด้านล่าง — ต้องรันทั้ง 2 ส่วน แต่ละส่วนแยกกัน
📌
room_number ในตาราง rooms_data เป็น text รองรับทั้งตัวเลข (1, 2, 10) และตัวอักษรผสม (A1, B2, 201) ได้สูงสุด 5 ตัวอักษร

4️⃣ ตั้งค่า Row Level Security (RLS)

RLS คือระบบป้องกันข้อมูล ทำให้เฉพาะผู้ที่ Login แล้วเท่านั้นที่เข้าถึงข้อมูลได้ ต้องรันกับ ทั้ง 2 ตาราง

-- เปิด RLS และตั้ง Policy สำหรับทั้ง 2 ตาราง
ALTER TABLE room_records ENABLE ROW LEVEL SECURITY;
CREATE POLICY "auth_users_only" ON room_records
FOR ALL USING (auth.role() = 'authenticated');

ALTER TABLE rooms_data ENABLE ROW LEVEL SECURITY;
CREATE POLICY "auth_users_only" ON rooms_data
FOR ALL USING (auth.role() = 'authenticated');
🔒
ถ้าไม่รัน SQL ส่วนนี้ ข้อมูลในฐานข้อมูลจะเป็น Public ทุกคนสามารถอ่านและแก้ไขได้ กรุณารันทุกครั้ง

5️⃣ สร้างบัญชีผู้ใช้สำหรับ Login

บัญชีนี้คือ Email/Password ที่จะใช้ Login เข้าเว็บออกใบเสร็จ (ไม่เกี่ยวกับบัญชี Supabase ที่สมัครไว้ในขั้นตอนที่ 1)

  1. ไปที่เมนู Authentication

    คลิกที่ไอคอน 🔑 Authentication ในเมนูซ้ายมือ

  2. คลิก "Users" → "Add user" → "Create new user"
  3. กรอก Email และ Password

    ใส่อีเมลและรหัสผ่านที่ต้องการใช้ Login (จดไว้กันลืม)

  4. เปิด "Auto Confirm User" ✅

    เปิดสวิตช์นี้ด้วย เพื่อให้ Login ได้ทันทีโดยไม่ต้องกดยืนยันอีเมล

  5. คลิก "Create user"
👥
ถ้ามีหลายคนช่วยกันออกใบเสร็จ สามารถทำซ้ำขั้นตอนนี้เพื่อสร้างหลายบัญชีได้ ทุกคนจะเห็นข้อมูลห้องเดียวกันหมด

6️⃣ ดึง API URL และ Anon Key

ค่า 2 อย่างนี้คือ "กุญแจ" ที่ใช้เชื่อมเว็บออกใบเสร็จเข้ากับฐานข้อมูลของคุณ ทั้งสองค่านี้ อยู่คนละหน้ากัน

📡 6A — หา API URL

⚙️ Project Settings › Integrations › Data API

ในหน้า Data API คัดลอกค่าใต้หัวข้อ "Project URL"

🚫
✅ ถูกต้อง: https://xxxxxxxxxxxxxxxx.supabase.co
❌ ผิด: https://xxxxxxxxxxxxxxxx.supabase.co/rest/v1/
❌ ผิด: https://xxxxxxxxxxxxxxxx.supabase.co/ (มี / ปิดท้าย)

🔑 6B — หา Anon Key

⚙️ Project Settings › API Keys › แท็บ "Legacy anon, service_role API keys"

คัดลอกคีย์ที่ชื่อ "anon" (public) เท่านั้น ขึ้นต้นด้วย eyJ...

🚫
ห้ามใช้คีย์ที่ชื่อ "service_role" เด็ดขาด! ให้ใช้แค่คีย์ "anon" เท่านั้น

7️⃣ กรอกข้อมูลเข้าเว็บออกใบเสร็จ

นำค่าที่ได้จากขั้นตอนที่ 6 มากรอกในเว็บออกใบเสร็จ

  1. เปิดเว็บออกใบเสร็จ (index.html)

    จะเจอหน้า Login

  2. คลิก "⚙️ ตั้งค่าฐานข้อมูล Supabase ของฉัน (สำหรับผู้ใช้ใหม่)"

    ปุ่มนี้อยู่ด้านล่างของหน้า Login ไม่ต้อง Login ก่อนก็กดได้

  3. วาง Supabase URL และ Anon Key

    นำค่าที่ได้จากขั้นตอน 6A และ 6B มาวางในช่องที่ตรงกัน

  4. ตรวจสอบ URL ก่อนบันทึก

    URL ต้องเป็น https://xxxxxxxxxxxxxxxx.supabase.co — ไม่มี /rest/v1/ และไม่มี / ปิดท้าย

  5. กด "บันทึกและกลับไปเข้าสู่ระบบ"

    ระบบจะพากลับไปหน้า Login อัตโนมัติ

  6. Login ด้วย Email/Password จากขั้นตอนที่ 5

    เข้าสู่ระบบสำเร็จ จากนั้นไปทำขั้นตอนที่ 8 และ 9 ต่อได้เลย 🎉

📱
ถ้าเปลี่ยนไปใช้เครื่องอื่น หรือเบราว์เซอร์อื่น จะต้องทำขั้นตอนนี้ใหม่ เพราะการตั้งค่าเก็บแยกตามเบราว์เซอร์

8️⃣ ตรวจสอบความถูกต้องอัตโนมัติ

หลังจากตั้งค่าเสร็จแล้ว ให้ตรวจสอบว่าทุกอย่างทำงานถูกต้องก่อนเริ่มใช้งานจริง กรอกข้อมูลด้านล่างแล้วกดปุ่มตรวจสอบได้เลย

🔍 ตรวจสอบการตั้งค่าอัตโนมัติ

ระบบจะดึงค่า URL และ Key ที่เคยบันทึกไว้ในเบราว์เซอร์มาให้อัตโนมัติ (ถ้ามี)

📊 ผลการตรวจสอบ:

รายการตรวจสอบสิ่งที่ตรวจสอบถ้าไม่ผ่าน
📡 การเชื่อมต่อ SupabaseURL และ Key ถูกต้องไหมกลับไปทำขั้นตอนที่ 6 ใหม่
📋 ตาราง room_recordsมีตารางนี้ในฐานข้อมูลไหมรัน SQL ส่วนที่ 1 ในขั้นตอนที่ 3
🏠 ตาราง rooms_dataมีตารางนี้ในฐานข้อมูลไหมรัน SQL ส่วนที่ 2 ในขั้นตอนที่ 3
🔒 Row Level SecurityRLS เปิดใช้งานแล้วไหมรัน SQL ในขั้นตอนที่ 4
👤 บัญชีผู้ใช้Email/Password ใช้ Login ได้ไหมกลับไปสร้าง User ในขั้นตอนที่ 5
🔐 สิทธิ์การเข้าถึงหลัง Login แล้วอ่าน/เขียนข้อมูลได้ไหมตรวจสอบ Policy ในขั้นตอนที่ 4

9️⃣ เพิ่มห้องเช่าในระบบ

หลังจากผ่านขั้นตอนที่ 8 แล้ว ระบบจะยังไม่มีห้องเช่าใดๆ เลย ต้องเพิ่มห้องเองใน Dashboard จัดการห้อง

🏠 วิธีเพิ่มห้องเช่าใน Dashboard

  1. ไปที่หน้า Dashboard จัดการห้อง

    กดปุ่ม "🏠 Dashboard จัดการห้อง" ที่อยู่ใน header ด้านบนขวาของหน้าออกใบเสร็จ (หลัง login แล้ว)

  2. กดปุ่ม "➕ เพิ่มห้องใหม่"

    ปุ่มนี้อยู่มุมบนขวาของหน้า Dashboard

    ➕ เพิ่มห้องใหม่
    ← ปุ่มนี้อยู่บน Dashboard
  3. กรอกเลขห้องและค่าเช่าเริ่มต้น

    ระบบจะแนะนำเลขห้องถัดไปให้อัตโนมัติ แก้ไขได้ รองรับทั้งตัวเลข (1, 2, 10) และตัวอักษรผสม (A1, B2) สูงสุด 5 ตัวอักษร

  4. กด "➕ เพิ่มห้อง" เพื่อบันทึก

    ห้องจะปรากฏในหน้า Dashboard ทันที ทำซ้ำขั้นตอนนี้เพื่อเพิ่มห้องทีละห้อง

  5. กรอกรายละเอียดห้องเพิ่มเติม (ถ้ามี)

    คลิกที่การ์ดห้องเพื่อเปิด Modal แก้ไข สามารถเพิ่มชื่อผู้เช่า เบอร์โทร เงินมัดจำ และอื่นๆ ได้ภายหลัง

💡
เพิ่มห้องแล้วกลับไปออกใบเสร็จได้เลย — หน้าออกใบเสร็จจะแสดงรายชื่อห้องที่คุณเพิ่มไว้ใน dropdown "ห้องที่" อัตโนมัติ ถ้าไม่เห็นให้กดปุ่ม "🔄 รีเฟรชห้อง" ที่แถบด้านบนของหน้าออกใบเสร็จ
🎉
พร้อมใช้งานแล้ว! เมื่อเพิ่มห้องเสร็จแล้ว ระบบพร้อมออกใบเสร็จได้ทันที

🔧 แก้ปัญหาที่พบบ่อย

ปัญหาสาเหตุที่เป็นไปได้วิธีแก้
กดเข้าสู่ระบบแล้วไม่ขึ้นอะไร / ค้าง ยังไม่ได้ตั้งค่า Supabase URL/Key กดปุ่ม "⚙️ ตั้งค่าฐานข้อมูล" ที่หน้า Login แล้วกรอกใหม่ตามขั้นตอน 6-7
เข้าสู่ระบบไม่สำเร็จ: Invalid login credentials อีเมล/รหัสผ่านผิด หรือยังไม่ได้เปิด Auto Confirm User ตรวจสอบใน Supabase → Authentication → Users ว่าสร้างบัญชีไว้แล้ว และสถานะเป็น Confirmed
บันทึก/โหลดข้อมูลไม่ได้ (Failed to fetch) URL ผิดรูปแบบ เช่น มี /rest/v1/ ต่อท้าย หรือมี / ปิดท้าย ไปที่หน้า Login กด "⚙️ ตั้งค่าฐานข้อมูล" แก้ URL ให้เหลือแค่ https://xxxx.supabase.co
Dropdown "ห้องที่" ว่างเปล่า ไม่มีตัวเลือก ยังไม่ได้เพิ่มห้องในระบบ ไปที่ Dashboard จัดการห้อง กดปุ่ม "➕ เพิ่มห้องใหม่" แล้วกลับมากดปุ่ม "🔄 รีเฟรชห้อง"
เปิดเว็บจากเครื่อง/เบราว์เซอร์อื่นแล้วต้องตั้งค่าใหม่ทุกครั้ง การตั้งค่าเก็บไว้ใน localStorage ของเบราว์เซอร์นั้นๆ เท่านั้น ทำตามขั้นตอนที่ 6-7 ซ้ำในเบราว์เซอร์ใหม่
หา Anon Key ไม่เจอ เข้าผิดเมนู หรือยังไม่ได้คลิกแท็บที่ถูกต้อง ไปที่ Project Settings › API Keys แล้วคลิกแท็บ "Legacy anon, service_role API keys"
ขั้นตอนที่ 8 แจ้งว่า "ไม่พบตาราง rooms_data" ยังไม่ได้รัน SQL ส่วนที่ 2 ในขั้นตอนที่ 3 กลับไปรัน SQL ส่วนที่ 2 ในขั้นตอนที่ 3 แล้วกดตรวจสอบใหม่
💬
ถ้าทำตามทุกขั้นตอนแล้วยังติดปัญหา ให้ใช้ ขั้นตอนที่ 8 ตรวจสอบอัตโนมัติ เพื่อดูว่าปัญหาอยู่ตรงไหน หรือลองเปิดเว็บในโหมดไม่ระบุตัวตน (Incognito) เพื่อตัดปัญหาเรื่องการตั้งค่าเก่าค้างอยู่
